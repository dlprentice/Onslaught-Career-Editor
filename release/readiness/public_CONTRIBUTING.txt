# Contributing From A Curated Source Candidate

This package is a reviewable source snapshot, not the complete public-primary
Git checkout. Read [AGENTS.md](AGENTS.md), [SECURITY.md](SECURITY.md), and
[CURRENT_CAPABILITIES.md](CURRENT_CAPABILITIES.md) before changing code. For
normal collaboration, branches, issues, and pull requests, use the canonical
public repository named in `EXPORT_PROVENANCE.json`.

## Boundaries

- Keep the installed game and original `BEA.exe` read-only.
- Patch or launch only copied targets and app-owned safe-copy roots.
- Do not add game binaries, archives, media, arbitrary saves/options, extracted
  assets, copied profiles, screenshots, raw debugger logs, full Ghidra stores,
  secrets, build output, or generated proof bundles.
- The canonical source repo retains one real game-generated regression save
  under a narrow maintainer exception. This curated candidate excludes it
  because immutability does not establish redistribution provenance;
  fixture-dependent UI tests skip when it is absent.
- Preserve unknown save/options bytes and do not synthesize `.bes` files from
  scratch.
- Keep Online Host/Join and parity claims bounded to accepted evidence.
- Do not add hosted CI or release automation; validation is local.

simulation truth in `OnslaughtRebuild.Core`; visual clients consume commands
and snapshots instead of owning gameplay state. The active implementation is
RE-informed original code. A strict clean-room claim requires separately
staffed specification, implementation, and acceptance teams.

## Local Validation

Run the smallest gate set matching the change, then the package-wide quick
check before handoff:

```powershell
npm test
```

For package-boundary sign-off, use the common block:

<!-- public-package-commands:start -->
```powershell
npm run test:doc-commands
npm run test:md-links
npm run test:winui-notices
npm run test:public-allowlist
npm run test:repo-hygiene
npm run test:rebuild-core
```
<!-- public-package-commands:end -->

Run `npm run test:public-candidate-inventory` only on a fresh materialized
candidate before build output exists. Run `npm run test:public-allowlist` at the
same pre-build point; filesystem payload mode intentionally rejects generated
`bin/obj` binaries. Migration inventory is source-repo-only because it requires
Git indexes and may compare a maintainer sibling checkout.

## Handoff Checklist

- Describe changed paths and user-visible behavior.
- Name exact validation commands and any intentionally skipped gate.
- Confirm no proprietary payload, secret, arbitrary save, screenshot, raw
  debugger output, or generated build artifact was added.
- Confirm the installed game and original executable were not mutated.
- Keep claims to what tests or bounded runtime evidence prove.
- Preserve the separate MIT toolkit and GPL rebuild license boundaries.

See [COLLABORATION.md](COLLABORATION.md) for the review handoff format and
[release/readiness/PUBLIC_SIGNOFF_COMMANDS.md](release/readiness/PUBLIC_SIGNOFF_COMMANDS.md)
for the wider local sign-off surface.
