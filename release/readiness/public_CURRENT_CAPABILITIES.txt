# Current Capabilities

Status: public-safe capability summary
Last updated: 2026-06-24

This package/export summary describes what outside contributors can rely on in
the public-primary source tree and downloadable release artifacts. The public
repo carries useful source, docs, tools, RE notes, state batons, agent reports,
readiness notes, and compact proof summaries. Hard payloads such as actual game
files, copied executables, raw saves/options, screenshots/frame dumps, raw CDB
logs, full Ghidra databases/backups, secrets, build output, and bulky generated
runtime captures remain local ignored overlays.

Public-source validation and downloadable release publication are separate
steps. The current downloadable app shape is an unsigned portable Windows x64
ZIP attached to GitHub Releases with a SHA-256 checksum sidecar. Signed
installer-grade MSIX, Store, SmartScreen/reputation, and trusted install remain
future proof, not current release claims.

## Product Surface

- WinUI 3 is the primary user-facing Windows app.
- AppCore backs save/options, patch planning, media/catalog, lore, and
  safe-copy workflows.
- The C# CLI is an active support surface for analyzer and patch-planning work.
- Current patch catalog accounting: 29 total rows; 20 visible options (9 stable, 11 experimental); 9 hidden companions.
- Windowed & Mods supports safe-copy patch planning, copied executable workflows,
  and a local split-screen launch preset for safe copies.
- Windowed & Mods shows sanitized safe-copy profile catalog provenance beside
  the selected preset, and packaged WinUI output includes both
  `patches/catalog/patches.v2.json` and
  `patches/catalog/safe-copy-profiles.v1.json`.
- Downloadable app releases are portable ZIPs. They include the WinUI app and
  third-party notices, not Battle Engine Aquila game files, saves, media, or
  private proof material.
- Online multiplayer is not available yet. Host/Join controls stay hidden or
  disabled and are never advertised as player-ready.
- Python under `tools/` is validation, release-policy, asset/RE, and lab
  tooling, not a product GUI lane.
- Electron, WPF, and the old Python GUI/CLI are archived/reference lanes.

For WinUI/AppCore work, also run the relevant build and test commands from
`PUBLIC_SIGNOFF_COMMANDS.md`.
For a fresh candidate intended for sharing, run
`npm run test:public-candidate-inventory` and `npm run test:public-allowlist`
before install/build/test outputs are created and verify
`EXPORT_PROVENANCE.json` is present. Payload mode intentionally rejects
generated `bin/obj` binaries; re-materialize a fresh candidate to repeat the
package-boundary proof after building.

## Claim Boundary

Public docs may summarize that the private static Ghidra work has bounded
function-quality and current-risk closure, but static closure is not runtime
gameplay proof, exact layout proof, patch behavior proof, visual proof, online
play, rebuild parity, or no-noticeable-difference parity.

Online multiplayer, matchmaking, native BEA netcode, active P3/P4 gameplay, and
player-ready separate-screen netplay are not public release capabilities yet.

## Safety Boundary

Public candidates must not include proprietary game assets, copied executable
bytes, saves, screenshots, frame captures, private proof bundles, local backup
paths, secrets, state files, or `subagents/` outputs. Patching and runtime work
must use copied executables or app-owned roots, not the installed Steam folder
or original `BEA.exe`.
