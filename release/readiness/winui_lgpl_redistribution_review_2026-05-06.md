# WinUI LGPL Redistribution Review - 2026-05-06

Status: public-safe release readiness evidence

## Scope

This note records the current redistribution posture for LGPL-bearing media dependencies in the WinUI-first product lane. It is not legal approval and it is not a signed-installer release claim. It exists to keep the binary-release boundary explicit before any public ZIP/MSIX/installer is cut.

This report is public-safe. It does not include private game paths, extracted game assets, screenshots, copied executables, runtime proof, raw NuGet package payloads, or local proof JSON.

| Package | Version | Why it matters | Current posture |
| --- | ---: | --- | --- |
| `LibVLCSharp` | `3.9.7.1` | Managed .NET binding used by the WinUI media player. | LGPL notice/relinkability review required before binary redistribution. |
| `VideoLAN.LibVLC.Windows` | `3.0.23.1` | Native LibVLC Windows runtime used by in-app Bink/video playback support. | LGPL notice/relinkability review required before binary redistribution. |

## Draft Notice Output

`release/readiness/THIRD_PARTY_NOTICES.winui-draft.md` is now generated by:

```powershell
py -3 tools\generate_winui_third_party_notices.py
```

Drift is checked by:

```powershell
npm run test:winui-notices
```

The notice draft is generated from the restored project dependency graph and local NuGet metadata. It intentionally omits local machine paths and private runtime evidence.

## Packaging Checklist

Before any public WinUI binary release:

1. Publish the exact candidate output.
2. Re-run the notice check against the restored/published dependency graph.
3. Include the notice draft or a final derivative in the package.
4. Include package-provided license/notice files required by redistributed dependencies.
5. Confirm the chosen distribution shape permits users to replace or relink LGPL-covered LibVLC components where required.
6. Avoid static modification of LGPL-covered binaries unless legal review explicitly approves the distribution obligations.
7. Keep private game assets, copied executables, screenshots, runtime proof, and local generated exports outside public release packages.

## What Is Proven

- The current WinUI product lane identifies LGPL-bearing media dependencies explicitly.
- A source-controlled third-party notice draft exists for the restored WinUI/support/test dependency graph.
- The notice draft can be regenerated and checked deterministically.
- Public release docs now have a concrete notice/check command instead of an untracked manual reminder.

## What Is Not Proven

- Final installer/ZIP/MSIX notice inclusion.
- Legal approval for LGPL compliance.
- User-facing relink/replacement instructions for a concrete binary package.
- Compatibility with upgraded LibVLCSharp or VideoLAN.LibVLC.Windows packages.
- Redistribution approval for any game-origin assets or private extracted media.

## Current Decision

The WinUI source candidate has a public-safe notice draft and LGPL redistribution checklist. Signed/installer-grade binary release remains blocked until final package output includes the required notices/license files and the LGPL strategy is reviewed against that exact output.
