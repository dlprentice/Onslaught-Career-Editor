# Public MSL Scripting Overview

Status: public-safe overview
Last updated: 2026-06-22

Battle Engine Aquila uses Mission Scripting Language (`.msl`) files for mission
logic, objectives, scripted messages, AI state changes, spawning, camera events,
and career-slot effects. This public overview keeps the useful language shape
available to contributors without copying private proof ledgers, backup paths,
or raw runtime evidence.

## Language Shape

- C-like syntax with functions, events, `if`, `switch`, `while`, and `for`.
- Common callbacks include `init()`, `started_dying()`, `died()`, and `hit(...)`.
- Event blocks use string names such as `event("game playing")`.
- `vars` / `end_vars` blocks declare mission-local values.
- Important value types include `int`, `float`, `bool`, `thing`, and `position`.
- `Pause(seconds)` is commonly used inside long-running script loops.

## Common Command Families

- Mission outcome: `LevelWon`, `LevelLostString`, objective complete/fail calls.
- Object references: `GetPlayer`, `GetThingRef`, `GetComponent`.
- Entity control: activation, shutdown, health, vulnerable state, AI state.
- Weapons and mode control: enable/disable weapons and flight mode.
- Messages and HUD: character messages, help messages, HUD highlight, variables.
- Spawning and movement: spawn helpers, waypoint following, teleport, positions.
- Career state: score, slots, and Goodie state helpers.

## Contributor Boundary

Public code may use this overview for parser tests, UI wording, safe tooling, and
clean-room planning. Do not submit proprietary mission scripts, extracted game
assets, private proof artifacts, screenshots, or copied game files.

Runtime MissionScript behavior, loose-vs-packed script selection, command
effects, visual HUD behavior, exact VM/layout details, patching behavior, rebuild
parity, and no-noticeable-difference parity are separate proof classes.
