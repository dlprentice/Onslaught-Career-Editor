## Purpose
Initializes the `CText` object by loading a language-specific text file. Handles multiple text file format versions (0, 1, 2, 3) and supports both regular and American English variants.

In the installed game, these text IDs are also referenced by mission scripts via `#define` constants in `game/data/MissionScripts/text/text.stf` (auto-generated by `TextConvert.cpp`).

## Signature
```c
// Thiscall convention (ECX = this)
void CText::Init(uint languageId);
```

## Parameters
- `languageId` (param_1): Language to load
  - 0 = English
  - 1 = French
  - 2 = German
  - 3 = Spanish
  - 4 = Italian

## Key Observations

### Language File Selection
The function constructs the path `data/LANGUAGE/<lang>.DAT` based on the language ID:
- `data/LANGUAGE/english.DAT`
- `data/LANGUAGE/french.DAT`
- `data/LANGUAGE/german.DAT`
- `data/LANGUAGE/spanish.DAT`
- `data/LANGUAGE/italian.DAT`

If `DAT_0083d990` is non-zero, uses American English: `data/LANGUAGE/american.DAT`

### Text File Format Versions
1. **Legacy (version 0):** First dword is string count, followed by offset table, then strings
2. **Version 1:** Magic `0xFFFFFFBB`, version=1, different table layout
3. **Version 2/3:** Magic `0xFFFFFFBB`, version=2 or 3, `count*0x0C` entry table + UTF-16 string pool and optional audio name pool

Note: in the file bytes, the magic appears as `bb ff ff ff` (little-endian `0xFFFFFFBB`).

#### Version 3 (Shipped Game Files)

All `game/data/language/*.dat` files in this repo are **version 3**:
- `ver_flags = 3` (high-bit clear)
- `count = 2571`

The observed v3 layout (matches `CText__Init` parsing):

| Offset | Type | Meaning |
|--------|------|---------|
| 0x00 | u32 | Magic = `0xFFFFFFBB` |
| 0x04 | u32 | `ver_flags` (v3 = `3`; bit `0x80000000` controls extra loader path; not set in shipped files) |
| 0x08 | u32 | `count` |
| 0x0C | Entry[count] | 0x0C bytes each: `{ u32 text_id; u32 text_off_words; u32 audio_off_bytes }` |
| 0x0C+count*0x0C | u32 | `uVar7` (language-specific offset used by the loader) |
| +4 | UTF-16LE pool | NUL-terminated strings; `text_off_words` is in UTF-16 code units from this pool start |
| (computed) | ASCII pool | Optional audio name pool; `audio_off_bytes` indexes into this pool; `0xFFFFFFFF` means “no audio” |

Entry fields:
- `text_id`: numeric ID used by scripts (`text/text.stf`) and game systems.
- `text_off_words`: offset measured in **UTF-16 code units** (multiply by 2 for byte offset).
- `audio_off_bytes`: byte offset into the audio-name pool; `0xFFFFFFFF` for entries with no voice.

Audio-name pool location (as in the loader):
- `iVar3 = uVar7 + count*0x0C`
- `audio_pool_size = *(u32*)(base + iVar3 + 0x10)`
- `audio_pool = base + iVar3 + 0x14`

In shipped files, `audio_pool_size` is consistently `0x2685` (9861 bytes) and the pool contains NUL-terminated ASCII identifiers like `HUD_01`, `003_WEST_SPHERE_B`, `WINGMAN_KILL_BILLY_1`.

- Logs "ERROR: Can't open text file %s" if file open fails
- Logs "unknown version in text file" for unrecognized format versions
