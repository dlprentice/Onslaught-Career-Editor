## Phase Rules

- Never mutate `C:\Program Files (x86)\Steam\steamapps\common\Battle Engine Aquila`, the installed `BEA.exe`, or any original source specimen in place.
- Player-facing patch work must use copied executables or playable copied game folders under an app-owned artifact root.
- Byte-patch proof, resource-mod proof, runtime launch/capture proof, clean-room rebuild proof, visual QA, and no-noticeable-difference parity are separate claims.
- Online multiplayer, in-game toggle menus, control-feel byte patches, and Enhanced profile presets must remain separate modules with their own proof levels, dependencies, conflicts, and restore boundaries.
- Product builds must load `patches/catalog/patches.v2.json`; built-in fallback patch rows are verification-only unless a test explicitly opts into fallback.
- If a local `BEA.exe` hash differs from the canonical clean hash, mutation can proceed only as an explicit byte-layout-only copied-target operation after selected patch offsets are verified as known original/already-patched states.
